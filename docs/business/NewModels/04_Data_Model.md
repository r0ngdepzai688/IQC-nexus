# 04. Data Model

Every entity in the New Models module must adhere to this data dictionary.

## Entities

### Project
- **Purpose:** Represents the introduction of a New Model.
- **Fields:** ID, Name, Launch Schedule, Current Stage (PV, PR, SR, MP), Status.
- **Owner:** Group Leader.
- **Lifecycle:** Created -> PV -> PR -> SR -> MP -> Closed.

### Model
- **Purpose:** The physical product being manufactured.
- **Fields:** Model Code, Description, Category.
- **Owner:** HQ/Admin.
- **Lifecycle:** Active -> EOL.

### Vendor
- **Purpose:** Supplier of materials.
- **Fields:** Vendor Code, Name, Risk Score, Approval Status.
- **Owner:** Part Leader.
- **Lifecycle:** Pending -> Auditing -> Approved / Rejected.

### Material
- **Purpose:** BOM component.
- **Fields:** Part Number, Description, Is_Grade_A, Category, Supplier_ID.
- **Owner:** Part Leader.
- **Lifecycle:** Identified -> Mapped -> Approved.

### Risk
- **Purpose:** Tracks potential issues.
- **Fields:** Title, Description, Classification (Low, Medium, High, Critical), Mitigation, Deadline.
- **Owner:** Assigned Assignee.
- **Lifecycle:** Identified -> Mitigating -> Closed.

### DFx
- **Purpose:** Design for Excellence document tracking.
- **Fields:** Document ID, Material_ID, Status (Pending, Reviewed, Completed).
- **Owner:** Staff / Part Leader.
- **Lifecycle:** Pending -> Reviewed -> Completed.

### Inspection Spec (ISS)
- **Purpose:** Standard for material inspection.
- **Fields:** Spec ID, Material_ID, Status.
- **Owner:** Staff.
- **Lifecycle:** Draft -> Review -> Released.

### Golden Sample
- **Purpose:** Physical reference sample.
- **Fields:** Sample Type (Golden, Limit, Color), Storage Location, Approval Status.
- **Owner:** Staff.
- **Lifecycle:** Requested -> Received -> Approved.

### Approval
- **Purpose:** Audit trail for sign-offs.
- **Fields:** Entity_ID, Entity_Type, Approver_ID, Status, Timestamp.
- **Owner:** System.
- **Lifecycle:** Pending -> Approved / Rejected.

### Issue
- **Purpose:** Tracks non-conformances or delays.
- **Fields:** Title, Description, Status, Assigned_To.
- **Owner:** Cell Leader.
- **Lifecycle:** Open -> In Progress -> Resolved.

### Task
- **Purpose:** Actionable item assigned to a user.
- **Fields:** Title, Due_Date, Assigned_To, Status.
- **Owner:** User.
- **Lifecycle:** To Do -> In Progress -> Done.

### Comment
- **Purpose:** Collaboration on entities.
- **Fields:** Content, Author_ID, Timestamp, Entity_ID.
- **Owner:** User.

### Notification
- **Purpose:** Alert generated by the system.
- **Fields:** Type, Message, Recipient_ID, Is_Read.
- **Owner:** System.

### KPI
- **Purpose:** Metric tracking.
- **Fields:** Metric_Name, Value, Target, Status (Green, Yellow, Red).
- **Owner:** System.
